ภาพรวม

เรากำลังสร้างระบบจองที่พักออนไลน์

ปัจจุบันลูกค้าทุกคนต้องทักแชทเข้ามาจอง ทีมต้องตอบทีละคน เช็คสลิปด้วยตา และทำใบเสร็จเองทุกครั้ง ระบบใหม่จะย้ายการจองทั้งหมดมาที่หน้าเว็บ ลดงานซ้ำของทีม และทำให้ลูกค้าจองเสร็จได้ในไม่กี่นาที

01
จองผ่านเว็บไซต์
ลูกค้ากดจองห้องได้จากหน้าเว็บโดยตรง เลือกวัน กรอกข้อมูล และยืนยันการจองในขั้นตอนเดียว
02
ยืนยันสลิปอัตโนมัติ
ลูกค้าสแกน QR และอัปโหลดสลิป ระบบเช็คยอดเงินกับธนาคารให้ทันที ไม่มีคนต้องเช็คมือ
03
แจ้งทีมงานทันที
ทุกการจองที่สำเร็จจะส่งเข้ากลุ่ม LINE ของทีมงานทันที พร้อมรายละเอียดครบ ไม่ต้องไล่ดูเอง
04
ดูรายได้ได้ทุกเวลา
หน้าหลังบ้านแสดงรายได้รายวัน สัปดาห์ เดือน พร้อมกราฟ การจอง และสถานะลูกค้าแบบรวมในที่เดียว

EASYSLIP API · QR + SLIP VERIFY

การชำระเงินและการตรวจสลิปอัตโนมัติ

QR Generation
POST /api/payments/qr
Input: booking_id, amount
Call: POST api.easyslip.com/v1/qr/generate
บันทึก: qr_payload + qr_expires_at ใน payments
Return: QR base64 → frontend แสดง
หมดอายุ 15 นาที ตรงกับเวลาที่ระบบล็อคห้องไว้ ถ้าลูกค้าไม่ชำระจะปลดล็อคให้คนอื่นจองต่อได้
Slip Verification
POST /api/payments/slip
• รับไฟล์สลิปจาก client (multipart)
• Upload ไป Supabase Storage payment-slips/
• เรียก EasySlip verify ด้วยรูปที่อัปโหลด
เช็ค 3 อย่าง:
a. amount ตรงกับยอด (±0 บาท)
b. receiver_account ตรงกับบัญชี LandCamp
c. slip_ref ยังไม่เคยใช้
• ถ้าผ่าน → booking.status = 'confirmed'
กลไกกันโกง
เก็บ slip_ref unique
สลิปเดียวกันใช้ซ้ำกับการจองอื่นไม่ได้
เช็คเวลาในสลิป
ต้องอยู่ในช่วง 24 ชั่วโมงล่าสุด
ตกหล่นไป Manual
ถ้า verify ไม่ผ่าน → payment_review ให้แอดมินเช็คมือ

ENVIRONMENT VARIABLES · ตัวแปรลับของระบบ

ค่าตั้งค่าที่ต้องเพิ่มใหม่

เก็บใน Vercel Environment Variables ตัวที่ขึ้นต้นด้วย NEXT_PUBLIC_ ใช้บน frontend ได้ ที่เหลือเป็นความลับใช้เฉพาะฝั่งเซิร์ฟเวอร์

# Supabase (มีอยู่แล้ว — เพิ่ม service role) SUPABASE_SERVICE_ROLE_KEY=... # Google OAuth (ตั้งใน Supabase Auth dashboard) NEXT_PUBLIC_GOOGLE_CLIENT_ID=... # EasySlip EASYSLIP_API_KEY=... EASYSLIP_BANK_ACCOUNT=... # บัญชีรับเงิน EASYSLIP_BANK_CODE=... # LINE OA LINE_CHANNEL_ACCESS_TOKEN=... LINE_CHANNEL_SECRET=... LINE_NOTIFY_GROUP_ID=... # group ID ของกลุ่มทีมงาน # Email RESEND_API_KEY=... EMAIL_FROM=booking@landcamp.com # App NEXT_PUBLIC_APP_URL=https://landcamp-eta.vercel.app BOOKING_CODE_PREFIX=LC # LC-2026-0001
วิธีจัดการ: ใช้คำสั่ง vercel env pull ดึงค่ามาที่เครื่องนักพัฒนา · ใช้ vercel env add เพิ่มค่าใหม่ทุก environment (preview / production) · ไม่ commit ค่าจริงลง git

แก้เว็บไซต์ได้เอง

เปลี่ยนรูป ราคา รายละเอียดห้อง เขียนบทความ SEO ได้จากหลังบ้านโดยตรง พร้อมตัวอย่างก่อนเผยแพร่แบบ Live Preview

ฐานข้อมูลลูกค้า
