# ☐ ส่วนที่ต้องทำ (เรียงตามลำดับที่ควรทำก่อน)

## ลำดับ 1 — อุดช่องโหว่ + เก็บงานค้าง (กำลังทำ)
> เล็กแต่สำคัญ ทำก่อนเพราะกระทบความปลอดภัย/ความน่าเชื่อถือ
- ☐ **1.1 ตั้งรหัสลับ `ADMIN_SESSION_SECRET` บน Vercel** — ✅ สร้างค่า + ใส่ `.env.local` แล้ว / ⏳ รอคุณเพิ่มบน Vercel
- ☐ **1.2 ตั้ง `CRON_SECRET` บน Vercel** — ✅ สร้างค่า + ใส่ `.env.local` แล้ว / ⏳ รอคุณเพิ่มบน Vercel
- ✅ **1.3 ลบรหัส super admin ออกจากไฟล์** (อ่านจาก env แทน) — ⏳ แนะนำเปลี่ยนรหัสจริงผ่าน `/admin/users`
- ✅ **1.4 แก้ชื่อห้องในประวัติจองของลูกค้า** — `/account/bookings` โชว์ชื่อห้องแล้ว
- ✅ **1.5 ย้ายรูปสลิปไปเก็บที่ Supabase Storage** — bucket `slips` (private) + signed URL ในหลังบ้าน
- ☐ **1.6 เคลียร์ตารางแอดมินเก่า (`admin_users`)** — ✅ เขียน migration 010 แล้ว / ⏳ รอคุณรันใน SQL Editor

## ลำดับ 2 — แจ้งเตือนลูกค้า + ทีมงาน (สำคัญสุดเชิงธุรกิจ)
> ตอนนี้จองเสร็จ "เงียบสนิท" ลูกค้าไม่ได้อะไร ทีมงานไม่รู้ — ช่องโหว่ใหญ่สุด
- ☐ **2.1 อีเมลยืนยันการจอง** (ใช้ Resend) — ต้องสมัคร Resend + ใส่ API key
- ☐ **2.2 แจ้งเตือนเข้ากลุ่ม LINE ทีมงาน** เมื่อมีจอง/แนบสลิป — ต้องมี LINE token + Group ID
- ☐ **2.3 ใบเสร็จ/ใบยืนยันการจองเป็น PDF** (ส่งแนบอีเมล + ดาวน์โหลดได้)
- ☐ **2.4 หน้าเช็คอินสำหรับพนักงาน** `/verify` — สแกน QR/รหัสจอง → กดยืนยันลูกค้ามาถึง

## ลำดับ 3 — ระบบแก้ไขเนื้อหาเว็บเอง (CMS) ⭐ ของที่อยากได้
> ตอนนี้ข้อความ/รูปทั้งเว็บฝังในโค้ด แก้ทีต้อง deploy ใหม่ — ทำระบบให้แก้เองได้ตลอด
- ✅ **3.1 สร้างที่เก็บเนื้อหา** — ตาราง `site_content` (ร่าง/เผยแพร่) + override ทับค่าเดิม เว็บไม่พังแม้ยังไม่แก้ / DB ล่ม (⏳ owner รัน migration 011)
- ◐ **3.2 หน้า `/admin/content`** — แก้ข้อความ 2 ภาษา + บันทึกร่าง + เผยแพร่ทันที ✅ (ตอนนี้เปิดให้แก้: เกี่ยวกับเรา, ส่วนติดต่อ, ข้อมูลติดต่อ, ท้ายเว็บ) / ☐ เหลือต่อ: Hero, ห้องพัก, เมนู, รีวิว, แกลเลอรี (รอ wire เพิ่ม)
- ☐ **3.3 จัดการรูป** — ลาก-วางอัป, สลับลำดับแกลเลอรี, เพิ่ม/ลบรูป
- ◐ **3.4 เผยแพร่ + ย้อนเวอร์ชัน** — กดเผยแพร่เว็บจริงอัปเดตทันทีไม่ต้อง deploy ✅ + เก็บประวัติทุกครั้งที่เผยแพร่ ✅ / ☐ เหลือ: หน้าดูตัวอย่างร่างสด + ปุ่มย้อนเวอร์ชัน

## ลำดับ 4 — CRM + ข้อมูลลูกค้า + Walk-in (ต่อ backend จากโครงที่ทำไว้)
- ☐ **4.1 โชว์รูป + อีเมลลูกค้าในหลังบ้าน** (ข้อมูลมีอยู่แล้วจากตอน login Google — แค่เอามาแสดง)
- ☐ **4.2 CRM เต็ม** — โน้ต, แท็ก VIP, ประวัติการติดต่อ, ยอดใช้จ่ายสะสม (ต่อจากหน้า CRM ที่ขึ้นโครงไว้)
- ☐ **4.3 Walk-in ใช้งานจริง** — พนักงานเพิ่มลูกค้า + จองแทน + กดว่าจ่ายแล้ว (ต่อ backend จากฟอร์มที่ทำไว้)

## ลำดับ 5 — ฟีเจอร์เพิ่มรายได้
- ☐ **5.1 ขายเสริม (Add-ons/Upsell)** ตอนจอง — อาหารเช้าพิเศษ, ชุด BBQ, เช็คเอาท์สาย, จัดวันเกิด
- ☐ **5.2 โค้ดส่วนลด + บัตรของขวัญ (Gift Voucher)**
- ☐ **5.3 ราคาตามฤดู/วันหยุด/พักยาว (Dynamic Pricing)**
- ☐ **5.4 Waitlist** — ลงชื่อรอเมื่อห้องเต็ม

## ลำดับ 6 — เครื่องมือบริหารจัดการ
- ☐ **6.1 KPI โรงแรมจริง** — อัตราเข้าพัก (Occupancy), ราคาเฉลี่ย (ADR), รายได้ต่อห้อง (RevPAR)
- ☐ **6.2 กระดานแม่บ้าน (Housekeeping)** — สถานะห้อง real-time (ว่าง/เข้าพัก/รอทำความสะอาด)
- ☐ **6.3 แจ้งซ่อม (Maintenance)** — สร้างงาน/มอบหมายช่าง/แจ้ง LINE
- ☐ **6.4 บันทึกการกระทำแอดมิน (Audit Log)** — ใครทำอะไรเมื่อไหร่

## ลำดับ 7 — ประสบการณ์ลูกค้า
- ☐ **7.1 ส่งอีเมลขอรีวิวอัตโนมัติหลังเช็คเอาท์** → ดึงรีวิวมาแสดงหน้าเว็บ
- ☐ **7.2 ลูกค้าแก้ไข/ยกเลิกการจองเอง** + คำนวณคืนเงินตามนโยบาย
- ☐ **7.3 ระบบลูกค้าประจำ (Loyalty)** — สะสมแต้ม/ส่วนลด

## ลำดับ 8 — เชื่อมแพลตฟอร์มจอง (Channel Manager)
- ☐ **8.1 ซิงก์ปฏิทินกับ Airbnb / Booking.com / Agoda (iCal)** — กันจองซ้ำข้ามช่องทาง (สำคัญถ้าขายหลายที่)

## ลำดับ 9 — เก็บงานก่อนเปิดใช้จริง (Launch)
- ☐ **9.1 จำกัดการกดถี่ (Rate limit) + กันบอท (BotID)** บนปุ่มยืนยันจอง
- ☐ **9.2 ตรวจความปลอดภัยรอบสุดท้าย (Security review)**
- ☐ **9.3 ขึ้น production จริง** + ทดสอบ + เปลี่ยนโดเมนใน Google OAuth & Supabase เป็น `landcampkhaoyai.com`
